# Documentation du projet

## Structure des dossiers

- `.github/workflows/` → fichiers YAML pour GitHub Actions  
- `scripts/` → scripts utiles pour le projet  
- `README.md` → présentation et badges  
- `CONTRIBUTING.md` → guide pour contribuer

## Comment contribuer

1. Créer une branche : `git checkout -b feature/nom-de-branche`  
2. Faire les modifications  
3. Committer : `git commit -m "type: description (#issue)"`  
4. Pousser la branche : `git push -u origin feature/nom-de-branche`  
5. Ouvrir une Pull Request

## Notes importantes

- Toujours travailler sur une **branche séparée** pour chaque issue  
- Vérifier que le **workflow GitHub Actions** passe avant de merger
